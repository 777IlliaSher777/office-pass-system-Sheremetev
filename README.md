# Office Visitor Pass System — Hexagonal Architecture
## Опис проєкту
Даний проєкт реалізує систему створення перепусток для відвідувачів офісу.
Користувач може створити перепустку для відвідувача, вказавши:- ім’я відвідувача;- дату відвідування;- час відвідування;- відповідальну особу.
Система виконує перевірку бізнес-правил та створює перепустку лише у випадку коректних даних.
---
## Реалізований use case
Use case:> створення перепустки для відвідувача з перевіркою дати, часу та відповідальної особи.
Основний сценарій роботи:1. Користувач вводить дані відвідувача.2. Система перевіряє:   - чи дата не є минулою;   - чи час входить у робочий графік;   - чи існує відповідальна особа.3. Якщо всі перевірки успішні — створюється перепустка.4. Перепустка зберігається у системі.
---
## Обрана архітектура
У проєкті використано **Hexagonal Architecture (Ports and Adapters)**.
---
## Чому було обрано саме цю архітектуру
Hexagonal Architecture дозволяє:- відокремити бізнес-логіку від технічних деталей;- ізолювати domain logic від інфраструктури;- легко замінювати реалізації адаптерів;- зробити систему більш зрозумілою та підтримуваною.
У даному проєкті бізнес-логіка не залежить від:- console input/output;- способу збереження даних;- зовнішніх технологій.
---

## Розподіл відповідальностей
### Domain layerМістить:- сутності системи;- бізнес-правила;- domain logic.
Приклади:- Pass- Visitor- ValidationService
---
### Application layerМістить use case системи.
Відповідає за:- виконання сценарію створення перепустки;- координацію роботи domain та repository.
---
### PortsМістять інтерфейси взаємодії із зовнішнім світом.
Наприклад:- PassRepository- EmployeeRepository
---
### Adapters / InfrastructureМістять технічну реалізацію системи.
Наприклад:- console input/output;- збереження даних у пам’яті;- реалізацію repository.
---
## Бізнес-правила
У проєкті реалізовано такі бізнес-правила:- неможливо створити перепустку на минулу дату;- час відвідування повинен бути у межах робочого графіка;- відповідальна особа повинна існувати у системі.
